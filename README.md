A Semester Project of Operating System course.
Its a MultiClient file server where two functional sides are utilized ( Client and Server ).
Client Side Functionalities:
== Connect to server using TCP/IP socket
Request file operations: 
Open File, Read File, Write to File, Close File, Delete File, List Files, Receive acknowledgment or error messages for each operation.

Server Side Functionalities:
== Accept and manage multiple client connections, Spawn threads or processes to handle concurrent requests, Use mutexes, semaphores, and reader-writer locks for synchronization, Communicate using IPC mechanisms (pipes, message queues, shared memory), Implement basic deadlock prevention and scheduling strategies, Simulate virtual memory with paging and page replacement algorithms.
